🔐 第 2 步：生成 GitHub Token（1 分钟）
════════════════════════════════════════════════════════════════════════════

1. 访问：https://github.com/settings/tokens/new

2. 填写：
   - Token name: claude-deployment
   - Expiration: 90 days
   - Scopes: 勾选 "repo"

3. 点击 "Generate token"

4. 复制生成的 token（重要！不会再显示）

5. 在 Windows 命令行运行：
   git config --global credential.helper wincred

✅ 完成第 2 步！

════════════════════════════════════════════════════════════════════════════

📤 第 3 步：推送代码（2 分钟）
════════════════════════════════════════════════════════════════════════════

在 PowerShell 或 Git Bash 中运行：

cd D:\Pheebs\labor-law-website

然后运行以下命令：

git remote set-url origin https://github.com/Phoebeiscool-lalala/labor-law-database.git
git branch -M main
git push -u origin main

当要求输入 "Password" 时：
→ 粘贴你刚才复制的 GitHub Token（不是密码！）

✅ 完成第 3 步！

🌐 第 4 步：在 Vercel 部署（1 分钟）

7. 点击 "Import"

9. 等待 2-3 分钟...

💡 遇到问题？
════════════════════════════════════════════════════════════════════════════

问题：Git 推送失败 - Authentication failed

解决：
1. 生成新的 GitHub Token（上面的链接）
2. 清除旧凭证：git credential-manager-core erase https://github.com
3. 重新运行 git push 命令
4. 粘贴新 token

问题：Vercel 部署失败

解决：
1. 打开 Vercel 控制面板
2. 选择你的项目 "labor-law-database"
3. 点击 "Deployments"
4. 查看最新部署的日志找出错误
